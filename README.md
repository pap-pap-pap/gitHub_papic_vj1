#Projekt
<h1>1. Što je Visual Studio Code i za što se najčešće koristi?<br></h1>
VS Code – integriran softver koji služi za debugging, dovršavanje koda, kontrolu verzija s gitom itd. Najčešće se koristi za web razvoj te za razvoj aplikacija i koda.
<h1>2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?<br></h1>
Panel se zove Explorer
<h1>3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?<br></h1>
Ekstenzije služe za dodavanje funkcionalnosti. Dodajemo ih klikom na Extension panel te upisom naziva extenzije
<h1>4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?<br></h1>
File > Open folder
<h1>5. Što se sve može raditi u Exploreru?<br></h1>
U Exploreru možeš pregledavati, kreirati, otvarati te uređivati mape i datoteke
<h1>6. Kako možeš brzo pronaći određenu datoteku u projektu?<br></h1>
CTRL + P možemo brzo pronaći određenu datoteku
<h1>7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?<br></h1>
CTRL + \ možemo gledati dvije datoteke u editoru
<h1>8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?<br></h1>
SHIFT + ALT + F možemo pokrenuti automatsko formatiranje koda
<h1>9. Što je version control i zašto se koristi?<br></h1>
Version control prati sve promjene u prokektu te se koristi za pregledu povijesti promjena te vraćanje starijih verzija koda
<h1>10. Kako se u VS Code-u otvara panel za Git (Source Control)?<br></h1>
CTRL + SHIFT + G možemo otvoriti panel za Git
<h1>11.  Što znače commit, push, pull?<br></h1>
commit - sprema lokalne promjene uz poruku
push - šalje commit promjene na GitHub
pull - preuzima GitHub promjene na računalo
<h1>12. Čemu služi commit poruka i kako treba izgledati?<br></h1>
Commit poruka služi za objašnjenje promjene te njegov razlog. Treba biti kratka i jasna
<h1>13.  Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?<br></h1>
Source Control panela > klikneš na datoteku > Diff, desni klik > Open Changes
<h1>14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?<br></h1>
CTRL + S > Source Control > Stage promjena (klik na +) > Upisati poruku > Commit > Push
<h1>15.  Kako se vraćaš na stariju verziju datoteke ako si pogriješio?<br></h1>
Source Control panel > View File History > odaberi stari commit > Restore/Revert
<h1>16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?<br></h1>
U Control Panelu možemo vidjeti sve promjene u datotekama
